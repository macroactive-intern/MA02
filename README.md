# MacroTargetCalculator

A Laravel REST API that calculates personalised daily calorie and macronutrient targets based on user biometrics, activity level, and goal. Uses the Mifflin-St Jeor equation for BMR, applies an activity multiplier for TDEE, and splits macros either by a preset ratio or a default algorithm.

---

## System Requirements

- PHP 8.2+
- Composer 2+
- SQLite (local development) or MySQL / PostgreSQL (production)

---

## Local Setup

```bash
# 1. Clone the repository
git clone https://github.com/macroactive-intern/MA02.git
cd MA02

# 2. Install dependencies
composer install

# 3. Create your environment file
cp .env.example .env

# 4. Generate the application key
php artisan key:generate

# 5. Run migrations
php artisan migrate

# 6. Start the development server
php artisan serve
```

The API will be available at `http://localhost:8000`.

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `APP_KEY` | Yes | Generated by `php artisan key:generate` |
| `APP_ENV` | Yes | `local` for development, `production` for live |
| `APP_DEBUG` | Yes | `true` locally, **must be `false` in production** |
| `DB_CONNECTION` | Yes | `sqlite` locally; `mysql` or `pgsql` in production |
| `CORS_ALLOWED_ORIGINS` | Yes | Comma-separated list of allowed origins — never use `*` in production |

| `LOG_LEVEL` | No | `debug` locally, `error` or `warning` in production |

> **Production note:** SQLite is not suitable for production. Configure `DB_CONNECTION`, `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, and `DB_PASSWORD` for your production database.

## Authentication

This API uses [Laravel Sanctum](https://laravel.com/docs/sanctum) token-based authentication.

### Get a token

Use `php artisan tinker` to create a user and issue a token:

```bash
php artisan tinker --execute="echo App\Models\User::create(['name'=>'Test','email'=>'test@example.com','password'=>bcrypt('password')])->createToken('dev')->plainTextToken;"
```

### Use the token

Include the token as a Bearer token on every request:

```
Authorization: Bearer <your-token>
Accept: application/json
```

## API Endpoints

All endpoints are prefixed with `/api` and require authentication.

### POST /api/macro-targets

Calculate and save macro targets for the authenticated user.

**Request body:**

```json
{
    "weight_kg": 80,
    "height_cm": 180,
    "age": 25,
    "sex": "male",
    "activity_level": "moderate",
    "goal": "maintain",
    "preset": "balanced"
}
```

| Field | Type | Values |
|-------|------|--------|
| `weight_kg` | numeric | 20–300 |
| `height_cm` | numeric | 100–250 |
| `age` | integer | 16–100 |
| `sex` | string | `male`, `female` |
| `activity_level` | string | `sedentary`, `light`, `moderate`, `active`, `very_active` |
| `goal` | string | `lose`, `maintain`, `gain` |
| `preset` | string (optional) | `balanced`, `high_protein`, `keto`, `low_carb` |

**Response — 201 Created:**

```json
{
    "data": {
        "id": 1,
        "weight_kg": 80,
        "height_cm": 180,
        "age": 25,
        "sex": "male",
        "activity_level": "moderate",
        "goal": "maintain",
        "preset": "balanced",
        "daily_calories": 2798,
        "protein_g": 210,
        "carbs_g": 280,
        "fat_g": 93,
        "created_at": "2026-06-16T00:00:00.000000Z"
    }
}
```

---

### GET /api/macro-targets

Retrieve the authenticated user's macro target history. Ordered newest first, paginated at 10 per page.

**Response — 200 OK:**

```json
{
    "data": [ ... ],
    "links": { ... },
    "meta": {
        "current_page": 1,
        "per_page": 10,
        "total": 25
    }
}
```

## Calculation Method

1. **BMR** — Mifflin-St Jeor equation
2. **TDEE** — BMR × activity multiplier (`sedentary` 1.2 → `very_active` 1.9)
3. **Goal adjustment** — `lose` −500 kcal, `maintain` ±0, `gain` +300 kcal
4. **Macros (default)** — protein 2g/kg, fat 25% of calories, carbs from remainder
5. **Macros (preset)** — calories split by ratio and converted using 4/4/9 kcal per gram

## HSTS (Production)

When deploying behind HTTPS, configure your web server (nginx/Apache) to send the `Strict-Transport-Security` header:

```
Strict-Transport-Security: max-age=31536000; includeSubDomains
```

Do not set HSTS in application middleware — it must only be sent over HTTPS connections, which the web server can enforce correctly.
